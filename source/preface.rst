为什么学习汇编语言？
========================

尽管高级语言（如 C、Rust、Go）已经极大地简化了软件开发，但理解汇编语言仍然具有重要意义：

- **深入理解计算机体系结构**：汇编语言直接映射到 CPU 指令，是理解 "程序是如何运行的" 的最佳途径。
- **性能调优**：分析编译器生成的汇编代码可以帮助发现性能瓶颈。
- **逆向工程与安全**：恶意软件分析、漏洞挖掘、CTF 竞赛等均离不开汇编基础。
- **操作系统与嵌入式开发**：启动代码、中断处理、上下文切换等底层任务常需汇编实现。

目标读者
============

本教程面向：

- 已有至少一门高级语言（C/C++ 尤佳）编程经验的开发者
- 计算机专业学生及爱好者
- 对底层系统编程感兴趣的读者

预备知识
============

阅读本书前，建议读者具备：

1. 基本的编程概念（变量、循环、函数、指针）
2. 计算机组成原理的基础知识（CPU、内存、寄存器等概念）
3. Linux 命令行基本操作（本书示例基于 Linux x86_64 环境）

环境说明
============

本书所有示例均基于 **Linux x86_64** 平台，使用 **NASM** 汇编器，**GCC (C)** 编译器作为与高级语言交互的参考工具。

.. note::

   本书主线使用 C 语言讲解汇编与高级语言的交互。如果你更习惯 C++（g++），只需注意名称修饰（name mangling）的区别——详见 :ref:`chapter-07-calling-c`。
