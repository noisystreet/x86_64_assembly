=========================
什么是汇编语言
=========================

从高级语言到机器码
========================

我们日常编写的大多数程序（C、Rust、Go 等）最终都要被翻译成 **机器码** —— 即 CPU 可直接执行的二进制指令。这一翻译过程通常分为两步：

1. **编译**：高级语言 → 汇编语言
2. **汇编**：汇编语言 → 机器码

汇编语言正是处在这两个步骤的中间位置：它用人类可读的助记符（如 ``mov``、``add``、``jmp``）代替了二进制的机器码，每条汇编指令基本与一条机器指令一一对应。

汇编语言的角色
================

既然高级语言更易编写和阅读，为什么还要学汇编？

- **理解计算机本质**：汇编让你直接面对 CPU 的指令集、寄存器和内存模型，这是理解 "程序到底怎么跑" 的最佳途径
- **性能调优**：分析编译器输出的汇编代码，可以识别冗余指令、发现缓存不友好等底层问题
- **逆向工程与安全**：漏洞挖掘、恶意软件分析、CTF 竞赛等均需汇编基础
- **底层系统编程**：启动代码（bootloader）、中断处理、任务切换等场景离不开汇编

汇编 vs 高级语言对比
======================

.. list-table::
   :header-rows: 1

   * - 维度
     - 高级语言
     - 汇编语言
   * - 抽象层级
     - 接近人类语言
     - 接近机器码
   * - 可移植性
     - 一套代码多平台
     - 绑定特定架构
   * - 性能控制
     - 依赖编译器优化
     - 精确控制每条指令
   * - 开发效率
     - 高
     - 低
   * - 调试难度
     - 相对容易
     - 需要理解底层细节

x86 架构演进简史
==================

x86_64 并非凭空出现，它经历了长达数十年的演进。了解这段历史有助于理解为什么寄存器叫 ``rax`` 而不是 ``a``、为什么 "64 位" 来自 AMD 而非 Intel。

.. list-table::
   :header-rows: 1

   * - 年代
     - 架构
     - 位数
     - 标志性变化
     - 寄存器命名示例
   * - 1978
     - Intel 8086
     - 16 位
     - 第一款 x86 CPU
     - ``ax``, ``bx``, ``cx``, ``dx``
   * - 1985
     - Intel 386
     - 32 位 (IA-32)
     - 扩展到 32 位，引入保护模式
     - ``eax``, ``ebx``（加 ``e`` 前缀）
   * - 2003
     - AMD64 (x86_64)
     - 64 位
     - AMD 率先扩展 64 位，新增 r8\~r15
     - ``rax``, ``rbx``（加 ``r`` 前缀）
   * - 至今
     - x86_64
     - 64 位
     - 各家持续扩展指令集（SSE, AVX, ...）
     - 兼容 16/32 位子寄存器

关键点：

- **向后兼容**：x86_64 CPU 仍能运行 16 位和 32 位的代码，这是 ``rax`` 的低 32 位可访问为 ``eax``、低 16 位为 ``ax`` 的原因
- **AMD vs Intel**：64 位扩展是 AMD 在 2003 年率先推出的（AMD64），Intel 后来在 2004 年采用，故称 x86_64

常见的汇编语法风格
====================

.. list-table::
   :header-rows: 1

   * - 风格
     - 汇编器
     - 平台
     - 特点
   * - NASM
     - NASM, YASM
     - Linux / Windows
     - Intel 语法，本书采用
   * - AT&T
     - GAS (GNU Assembler)
     - Linux / Unix
     - 操作数顺序相反，需加 ``%``, ``$`` 前缀
   * - MASM
     - MASM (Microsoft)
     - Windows
     - Intel 语法，与 NASM 略有差异

本书所有示例均使用 **NASM** 汇编器，采用 Intel 语法风格。
