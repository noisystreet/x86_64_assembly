=========================
什么是汇编语言
=========================

从高级语言到机器码
========================

我们日常编写的大多数程序（C、Rust、Go 等）最终都要被翻译成 **机器码** —— 即 CPU 可直接执行的二进制指令。这一翻译过程通常分为两步：

1. **编译**：高级语言 → 汇编语言
2. **汇编**：汇编语言 → 机器码

汇编语言正是处在这两个步骤的中间位置：它用人类可读的助记符（如 ``mov``、``add``、``jmp``）代替了二进制的机器码，每条汇编指令基本与一条机器指令一一对应。

汇编语言的角色
================

既然高级语言更易编写和阅读，为什么还要学汇编？

- **理解计算机本质**：汇编让你直接面对 CPU 的指令集、寄存器和内存模型，这是理解 "程序到底怎么跑" 的最佳途径
- **性能调优**：分析编译器输出的汇编代码，可以识别冗余指令、发现缓存不友好等底层问题
- **逆向工程与安全**：漏洞挖掘、恶意软件分析、CTF 竞赛等均需汇编基础
- **底层系统编程**：启动代码（bootloader）、中断处理、任务切换等场景离不开汇编

汇编 vs 高级语言对比
======================

.. list-table::
   :header-rows: 1

   * - 维度
     - 高级语言
     - 汇编语言
   * - 抽象层级
     - 接近人类语言
     - 接近机器码
   * - 可移植性
     - 一套代码多平台
     - 绑定特定架构
   * - 性能控制
     - 依赖编译器优化
     - 精确控制每条指令
   * - 开发效率
     - 高
     - 低
   * - 调试难度
     - 相对容易
     - 需要理解底层细节

x86 架构演进简史
==================

x86_64 并非凭空出现，它经历了长达四十多年的演进。了解这段历史有助于理解：为什么寄存器叫 ``rax`` 而非 ``a``、为什么 "64 位" 来自 AMD 而非 Intel、以及向后兼容性如何造就了今天复杂的指令集。

.. list-table::
   :header-rows: 1

   * - 年代
     - 架构/微架构
     - 位数
     - 标志性变化
     - 寄存器命名
   * - 1978
     - Intel 8086
     - 16 位
     - 第一款 x86 CPU，奠定指令集基础
     - ``ax, bx, cx, dx``
   * - 1982
     - Intel 80286
     - 16 位
     - 引入保护模式（实模式 + 保护模式）
     - 同 8086
   * - 1985
     - Intel 80386 (IA-32)
     - 32 位
     - 扩展到 32 位，引入分页和虚拟内存
     - ``eax, ebx``（加 ``e`` 前缀）
   * - 1989
     - Intel 80486
     - 32 位
     - 集成 L1 缓存和 FPU，引入流水线
     - 同 386
   * - 1993
     - Intel Pentium
     - 32 位
     - :strong:`超标量` （双流水线），分支预测
     - 同 386
   * - 1995
     - Intel Pentium Pro
     - 32 位
     - :strong:`乱序执行` （P6 微架构），引入 µop
     - 同 386
   * - 1997
     - Intel Pentium II / III
     - 32 位
     - 引入 :strong:`MMX` / :strong:`SSE` 指令集
     - 同 386
   * - 2000
     - Intel Pentium 4 (NetBurst)
     - 32 位
     - 超深流水线（20+ 级），:strong:`SSE2`
     - 同 386
   * - 2003
     - AMD64 (K8)
     - :strong:`64 位`
     - AMD 率先扩展 64 位，新增 r8\~r15
     - ``rax, rbx``（加 ``r`` 前缀）
   * - 2004
     - Intel EM64T
     - 64 位
     - Intel 采纳 AMD64，双方兼容
     - 同 AMD64
   * - 2006
     - Intel Core (Conroe)
     - 64 位
     - 回归高效微架构，:strong:`SSSE3`
     - 同 AMD64
   * - 2008
     - Intel Nehalem (Core i)
     - 64 位
     - 集成内存控制器/GPU，:strong:`SSE4`
     - 同 AMD64
   * - 2011
     - Intel Sandy Bridge
     - 64 位
     - 引入 :strong:`AVX` （256 位 YMM 寄存器）
     - 同 AMD64
   * - 2013
     - Intel Haswell
     - 64 位
     - :strong:`AVX2`、BMI 位操作扩展、FMA
     - 同 AMD64
   * - 2015
     - Intel Skylake
     - 64 位
     - :strong:`ADX` （大整数扩展）、MPX
     - 同 AMD64
   * - 2017
     - Intel Skylake-SP / Xeon Phi
     - 64 位
     - :strong:`AVX-512` （512 位 ZMM + 掩码寄存器）
     - 同 AMD64
   * - 至今
     - x86_64（各家持续迭代）
     - 64 位
     - AMX（矩阵扩展）、APX 等
     - 兼容 16/32/64 位子寄存器

.. note::

   从这张时间线可以看出，x86_64 的核心设计理念是 :strong:`向后兼容`——40 多年前为 8086 编写的代码（只要不涉及已废弃指令），在今天最新的 CPU 上仍可运行。这也是 ``rax`` 的低 32 位可访问为 ``eax``、低 16 位可访问为 ``ax`` 的根本原因。

    另一个值得注意的转折点是 :strong:`AMD 主导 64 位扩展`：Intel 原本计划另起炉灶搞 IA-64（安腾），AMD 则务实地在 x86 基础上直接扩展到 64 位。市场选择了 AMD64，Intel 后来不得不放弃 IA-64 并采纳 AMD64。

常见的汇编语法风格
====================

.. list-table::
   :header-rows: 1

   * - 风格
     - 汇编器
     - 平台
     - 特点
   * - NASM
     - NASM, YASM
     - Linux / Windows
     - Intel 语法，本书采用
   * - AT&T
     - GAS (GNU Assembler)
     - Linux / Unix
     - 操作数顺序相反，需加 ``%``, ``$`` 前缀
   * - MASM
     - MASM (Microsoft)
     - Windows
     - Intel 语法，与 NASM 略有差异

本书所有示例均使用 **NASM** 汇编器，采用 Intel 语法风格。
