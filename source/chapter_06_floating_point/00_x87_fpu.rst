===============================
x87 FPU：浮点运算的起源
===============================

在 SSE 成为 x86_64 上浮点运算的主流方案之前，x87 FPU（Floating-Point Unit）
统治了 x86 的浮点世界近 20 年。了解 x87 不仅仅是看一段历史——它解释了为什么
C 语言的 ``long double`` 在 x86_64 上是 80 位（而非 64 位），也解释了为什么
SSE 的寄存器是 **平面化** 而非 **栈式** 的。

.. admonition:: 从独立芯片到集成 FPU 的进化史
   :class: story

   Intel 8086/8088（1978/1979）根本没有浮点运算能力——所有浮点计算都需要软件模拟，
   或者买一块单独的 **Intel 8087** 协处理器芯片。8087 是一个独立的芯片，插在 CPU 旁边的
   专用插座上，售价高达 **300 美元** （相当于今天的 ~1200 美元）。CPU 遇到浮点指令时，
   自动将指令发送给 8087 执行。这听起来很美好，但有一个大问题：**如果用户没买 8087，
   程序一执行浮点指令就会崩溃**。

   80287 和 80387 延续了这个模式，直到 :strong:`Intel 486DX（1989）` 才首次将 FPU 集成到
   CPU 内部。但 Intel 为了压低成本推出了砍掉 FPU 的 :strong:`486SX`——想用浮点的话还是得买
   487SX 协处理器。令人哭笑不得的是，487SX 其实是一颗完整的 486DX（含 FPU），
   插入后会:strong:`禁用原 486SX`，接管所有工作。直到 :strong:`Pentium（1993）`，FPU 才成为 x86 CPU
   的标准组成部分，协处理器的时代正式终结。

x87 寄存器栈
================

x87 的设计最独特之处在于它使用了一个 :strong:`8 深度的浮点寄存器栈` （ST0\~ST7），
而不是像通用寄存器那样随机访问。

.. code-block:: text

   ; x87 寄存器栈（栈顶 = ST0）
   ;
   ;  压入 (fld)         弹出 (fstp)
   ;      ↓                  ↑
   ;   ┌──────┐  ← ST0 (栈顶)
   ;   │      │
   ;   ├──────┤
   ;   │      │  ← ST1
   ;   ├──────┤
   ;   │ ...  │
   ;   ├──────┤
   ;   │      │  ← ST7 (栈底)
   ;   └──────┘

堆栈设计意味着：``fld``（加载）将值压入栈顶（原 ST0 下沉到 ST1），
``fstp``（存储并弹出）将栈顶写出后弹出。算术指令 ``fadd``、``fmul`` 等
默认操作 ST0 和 ST1，结果放在 ST0。

.. warning::

   x87 的栈式设计与通用寄存器完全不同，是造成 x87 难用的主要原因。
   编译器必须跟踪每个值的栈深度，这让 x87 的代码生成比 SSE 复杂得多。
   这也是 SSE 选择平面寄存器模型（16 个 XMM 寄存器，随机访问）的核心原因之一。

x87 浮点格式：80 位扩展精度
================================

x87 内部使用 :strong:`80 位扩展双精度` （1 位符号 + 15 位指数 + 64 位尾数），
比 IEEE 754 标准要求的 64 位双精度更宽：

.. list-table::
   :header-rows: 1

   * - 格式
     - 总位宽
     - 指数位
     - 尾数位
     - 精度（十进制）
   * - 单精度（float）
     - 32
     - 8
     - 23
     - ~7 位
   * - 双精度（double）
     - 64
     - 11
     - 52
     - ~15 位
   * - x87 扩展精度（long double）
     - **80**
     - **15**
     - **64**
     - **~19 位**

.. admonition:: C 语言中的 long double
   :class: application

   这就是为什么 GCC 在 x86_64 上 ``sizeof(long double)`` 是 **16 字节** （虽然只用了
   10 字节，2 字节填充），而 ``sizeof(double)`` 只有 8 字节。x87 始终用 80 位做中间
   计算，这意味着连续计算 ``a * b + c`` 在 x87 上比 SSE 上 **精度更高** （SSE 在每次
   运算后都要舍入到 64 位）。但也因此，x87 程序的运行结果可能因编译器优化等级不同而产生
   细微差异——这就是著名的 **x87 中间精度问题** 。

x87 指令集
===============

x87 的指令以 ``f`` 开头，按功能分类：

数据传送
--------------

.. code-block:: none

   fld    dword [val32]     ; 加载 32 位 float 到 ST0
   fld    qword [val64]     ; 加载 64 位 double 到 ST0
   fld    tword [val80]     ; 加载 80 位 long double
   fldz                     ; 加载 0.0 到 ST0
   fld1                     ; 加载 1.0 到 ST0
   fldpi                    ; 加载 π（圆周率）到 ST0

   fst    dword [val32]     ; 将 ST0 存储为 32 位 float（不弹出）
   fstp   qword [val64]     ; 将 ST0 存储为 64 位 double 并弹出
   fstp   tword [val80]     ; 存储 80 位并弹出

   fxch                     ; 交换 ST0 和 ST1

算术运算
--------------

.. code-block:: none

   ; 加法
   fadd                     ; ST0 = ST0 + ST1（栈顶两个相加）
   fadd  ST0, ST1           ; 同上（显式写法）
   fadd  ST0, ST2           ; ST0 = ST0 + ST2
   faddp                     ; ST1 = ST1 + ST0，然后弹出（结果在 ST1）
   fiadd dword [int32]      ; ST0 += *int32（整数操作数）

   ; 减法
   fsub                     ; ST0 = ST0 - ST1
   fsubp                     ; ST1 = ST1 - ST0，弹出
   fsubr                    ; ST0 = ST1 - ST0（反向减法）
   fisub dword [int32]

   ; 乘法
   fmul                     ; ST0 = ST0 * ST1
   fmulp                    ; ST1 = ST1 * ST0，弹出
   fimul dword [int32]

   ; 除法
   fdiv                     ; ST0 = ST0 / ST1
   fdivp                    ; ST1 = ST1 / ST0，弹出
   fdivr                    ; ST0 = ST1 / ST0（反向除法）
   fidiv dword [int32]

   ; 其他
   fsqrt                    ; ST0 = sqrt(ST0)
   fchs                     ; ST0 = -ST0（符号取反）
   fabs                     ; ST0 = |ST0|（绝对值）
   frndint                  ; ST0 = round(ST0)（四舍五入到整数）

比较运算
--------------

.. code-block:: none

   fcom                     ; 比较 ST0 和 ST1，设置状态字
   fcomp                    ; 比较 ST0 和 ST1 并弹出
   fcompp                   ; 比较 ST0 和 ST1 并弹出两次
   fcomi                    ; 比较 ST0 和 ST1 并设置 EFLAGS

   ; x87 的浮点比较需要先写回状态字再测试
   fcompp                   ; 比较 ST0 和 ST1，弹出两次
   fstsw  ax                ; 将 FPU 状态字写入 AX
   sahf                     ; 将 AH 的低 5 位复制到 EFLAGS（SF, ZF, AF, PF, CF）
   jb   .below              ; 如果 ST0 < ST1 则跳转（无符号低于）

   ; 更现代的方式（P6 引入）——直接设置 EFLAGS
   fcomip                   ; 比较 ST0 和 ST1，设置 EFLAGS，弹出
   ja   .above              ; 可以直接用有符号/无符号跳转

x87 与 SSE 对比
====================

.. list-table::
   :header-rows: 1

   * - 维度
     - x87
     - SSE
   * - 寄存器模型
     - 栈（8 深度 ST0-ST7）
     - 平面（16 个 XMM）
   * - 精度
     - 内部 80 位，有中间精度问题
     - 严格 IEEE 754，无中间精度
   * - 并行能力
     - 标量（一次一条浮点指令）
     - 向量（一次 2/4/8 个操作数）
   * - 现代编译器使用
     - 几乎不用（仅 ``long double``）
     - 默认浮点方案
   * - 适用场景
     - 需要 80 位精度的数值计算
     - 通用浮点、SIMD 并行

x87 编程示例
=================

.. code-block:: none

   ; 计算圆柱体积：V = π * r^2 * h
   ; 输入：xmm0 = r，xmm1 = h（为简化，此处用内存传参）
   section .data
       r  dq 2.5                     ; 半径
       h  dq 10.0                    ; 高度

   section .text
       fld1                          ; ST0 = 1.0
       fld   qword [r]               ; ST0 = r,   ST1 = 1.0
       fmul  ST0, ST0                ; ST0 = r^2, ST1 = 1.0
       fldpi                         ; ST0 = π,   ST1 = r^2, ST2 = 1.0
       fmul                          ; ST0 = π * r^2, ST1 = 1.0
       fmul  qword [h]               ; ST0 = π * r^2 * h, ST1 = 1.0
       ; 结果在 ST0

.. note::

   注意 x87 编程的特点：所有操作都在栈上完成，需要时刻跟踪每个值在栈中的位置。
   上面的例子中 ``fld1`` 提前压入 1.0 只是为了后续的栈平衡——这在 x87 编程中很常见。

x87 状态与控制字
====================

x87 FPU 有一个 16 位的 **状态字（Status Word）** 和一个 16 位的 **控制字（Control Word）**：

.. code-block:: none

   ; 状态字（FPU Status Word）
   ; bit 0-3: 异常标志
   ; bit 6-7: 栈顶指针（TOP）
   ; bit 8:   C0（比较结果位）
   ; bit 9:   C1
   ; bit 10:  C2
   ; bit 14:  C3（与 C0/C2 共同组成比较结果）

   ; 控制字（FPU Control Word）
   ; bit 8-9: 精度控制（00=24 位，01=53 位，10=64 位）
   ; bit 10-11: 舍入控制（00=最近, 01=向下, 10=向上, 11=截断）

   ; 设置 FPU 为 64 位精度（80 位中间计算）
   finit              ; 初始化 FPU（设置为默认状态）
   fldcw [cw_val]     ; 加载自定义控制字

.. admonition:: x87 的遗产
   :class: story

   尽管 SSE 已经完全取代了 x87，但 x87 留下了一个重要的遗产：**80 位扩展精度**。
   当你在 x86_64 Linux 上运行用到 ``long double`` 的程序时，编译器生成的代码
   实际上是在调用 x87 指令（因为 SSE 不支持 80 位）。这也意味着如果你在内核模块
   或信号处理函数中使用了 ``long double``，可能需要 **finit** 或 **fninit**
   来初始化 FPU——否则 x87 可能处于未定义状态，导致神秘崩溃。
   Linux 内核会在进程切换时保存/恢复 x87 状态（通过 ``fxsave``/``fxrstor``）。
