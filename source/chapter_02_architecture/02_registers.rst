===============================
寄存器
===============================

寄存器是 CPU 内部的高速存储单元，读写速度远快于内存。x86_64 架构提供了 40 多个寄存器，按用途分为几类。

通用寄存器
==============

x86_64 有 16 个通用寄存器（General-Purpose Registers, GPRs），每个都是 64 位宽。

.. admonition:: 为什么是 16 个寄存器？
   :class: story

   1978 年的 Intel 8086 只有 8 个 16 位寄存器（``ax``, ``bx``, ``cx``, ``dx``, ``si``, ``di``, ``bp``, ``sp``），
   这在当时已经足够。32 位时代（386）将这些寄存器扩展到了 ``e`` 前缀的 32 位版本。
   到了 64 位时代，AMD 不仅将寄存器扩到 64 位（``rax`` 等），还额外新增了 8 个通用寄存器（``r8``-``r15``）。
   这是 x86 历史上第一次真正增加通用寄存器的数量——对编译器而言，16 个寄存器比 8 个意味着更少的栈溢出，
   显著提升了性能。

.. list-table::
   :header-rows: 1

   * - 寄存器
     - 用途
     - 是否需保存（调用约定）
   * - ``rax``
     - 累加器；函数返回值
     - 调用者保存
   * - ``rbx``
     - 通用
     - **被调用者保存**
   * - ``rcx``
     - 循环计数；第 4 个参数
     - 调用者保存
   * - ``rdx``
     - 数据寄存器；第 3 个参数
     - 调用者保存
   * - ``rsi``
     - 源指针；第 2 个参数
     - 调用者保存
   * - ``rdi``
     - 目标指针；第 1 个参数
     - 调用者保存
   * - ``rbp``
     - 基址指针（栈帧）
     - **被调用者保存**
   * - ``rsp``
     - 栈指针
     - **被调用者保存**
   * - ``r8`` - ``r15``
     - 通用（新增的 64 位寄存器）
     - 调用者保存（r8-r11）
       :strong:`被调用者保存`\ （r12-r15）

子寄存器（Sub-Registers）
============================

x86_64 的向后兼容设计使得可以访问每个通用寄存器的低 32 位、16 位和 8 位部分：

.. code-block:: none

   ; 以 rax 为例的层级结构
   ; ┌──────────────────────────────────────────────────┐
   ; │                     rax (64 位)                   │
   ; ├──────────────────────┬───────────────────────────┤
   ; │        (高 32 位)     │        eax (32 位)         │
   ; │                      ├─────────────┬─────────────┤
   ; │                      │    ax (16)   │             │
   ; │                      ├──────┬──────┤             │
   ; │                      │ ah   │ al   │             │
   ; │                      │ (8)  │ (8)  │             │
   ; └──────────────────────┴──────┴──────┴─────────────┘

.. code-block:: none

   ; 实际操作示例
   mov rax, 0x1234567890ABCDEF  ; rax = 0x1234567890ABCDEF
   mov eax, 0xFFFFFFFF          ; 写入 eax 自动清零高 32 位！
                                 ; rax = 0x00000000FFFFFFFF
   mov ax,  0x1234              ; 只影响低 16 位
                                 ; rax = 0x0000000000001234
   mov al,  0x56                ; 只影响低 8 位
                                 ; rax = 0x0000000000000056
   mov ah,  0x12                ; 只影响第 8-15 位
                                 ; rax = 0x0000000000001256

.. warning::

   在 x86_64 模式下，**写入 32 位子寄存器（如 ``eax``）会自动将对应 64 位寄存器的高 32 位清零**。
   这是 x86_64 特有的行为，用于消除对高 32 位的隐式依赖。但写入 16 位或 8 位子寄存器不会影响高 48 位。

.. admonition:: 32 位清零高 32 位的秘密
   :class: story

   这个看似奇怪的设计是 Intel 在引入 x86_64 时做的一个精巧优化。在 32 位模式下，程序员经常写 ``mov eax, 0``
   来清零整个 ``eax`` （高 32 位不存在）。到了 64 位模式，如果写入低位需要等待高位就绪，会产生不必要的数据依赖。
   Intel 的解决方案是：**写入 32 位子寄存器时自动清零高 32 位**，这样 CPU 就知道新值不依赖旧的高位，
   可以进行寄存器重命名（register renaming），消除等待依赖。这是 "free zero" 优化的硬件基础。

专用寄存器
==============

指令指针（RIP）
------------------

``rip``（Instruction Pointer）指向当前正在执行的指令地址。程序无法直接修改 ``rip``，只能通过跳转指令（``jmp``、``call`` 等）间接改变。

.. code-block:: none

   ; RIP 相对寻址（x86_64 特性）
   lea rax, [rip + symbol]  ; 获取 symbol 相对于当前指令的地址

标志寄存器（RFLAGS）
-----------------------

``rflags`` 是一个 64 位寄存器，但只使用了低 32 位。每个位表示 CPU 的当前状态：

.. list-table::
   :header-rows: 1

   * - 标志
     - 位
     - 说明
     - 用途
   * - CF
     - 0
     - 进位标志
     - 无符号运算溢出
   * - PF
     - 2
     - 奇偶标志
     - 结果的低 8 位中有偶数个 1
   * - ZF
     - 6
     - 零标志
     - 结果为 0
   * - SF
     - 7
     - 符号标志
     - 结果为负数
   * - OF
     - 11
     - 溢出标志
     - 有符号运算溢出
   * - DF
     - 10
     - 方向标志
     - 字符串操作方向（递增/递减）

.. code-block:: none

   ; 标志位在条件跳转中的应用
   cmp rax, rbx      ; 比较 rax 和 rbx，设置标志位
   je  .equal         ; ZF=1 时跳转（相等）
   jg  .greater       ; SF=OF 且 ZF=0 时跳转（大于）
   jl  .less          ; SF!=OF 时跳转（小于）
   jge .ge            ; SF=OF 时跳转（大于等于）

段寄存器
============

x86_64 中段寄存器主要用于系统编程，用户态程序很少直接操作：

.. list-table::
   :header-rows: 1

   * - 寄存器
     - 用途
   * - ``cs``
     - 代码段
   * - ``ds``
     - 数据段
   * - ``ss``
     - 栈段
   * - ``es``, ``fs``, ``gs``
     - 附加段（\ ``fs`` 常用于线程本地存储 TLS）

.. note::

   在 64 位模式下，段寄存器的角色已被极大简化。大多数段基址被强制为 0，
   只有 ``fs`` 和 ``gs`` 仍然可以通过 MSR（Model-Specific Register）设置非零基址，
   用于实现线程本地存储（TLS）。

.. admonition:: 段寄存器的历史包袱
   :class: story

   x86 的段寄存器设计是 16 位时代的遗留品。8086 时代地址总线 20 位，但寄存器只有 16 位，
   为了访问 1 MB 内存，Intel 发明了"段：偏移"寻址：``物理地址 = 段 << 4 + 偏移``。
   这导致了许多奇怪的程序设计（如一个指针需要 32 位：16 位段 + 16 位偏移）。
   到了 64 位模式，段寄存器被大幅弱化——平坦内存模型让段基址固定为 0，
   唯一保留作用的是 ``fs``/``gs``，用于实现 TLS（线程本地存储）。
   Windows 用 ``gs``，Linux 用 ``fs``，这也是一种历史趣闻。
