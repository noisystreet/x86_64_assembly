===============================
内存模型
===============================

理解 x86_64 的内存模型是编写正确汇编代码的前提。本节介绍虚拟地址空间、内存布局、字节序和对齐规则。

虚拟内存地址空间
====================

Linux x86_64 使用 48 位虚拟地址空间（可扩展到 57 位）。用户态程序看到的是虚拟地址，由操作系统和 MMU（内存管理单元）映射到物理内存。

.. mermaid::

   flowchart TD
       subgraph User[用户空间 低 128 TB]
           Text[代码段 .text] --> Data[数据段 .data + .bss]
           Data --> Heap[堆 向上增长]
           Heap --> Gap[⬆ 空闲区域 ⬇]
           Gap --> Stack[栈 向下增长]
       end
       subgraph Kernel[内核空间 高 128 TB]
           KMem[内核代码与数据]
       end

.. note:: 典型程序入口地址为 ``0x400000``\ ，栈底在高地址、向低地址增长。

程序的内存布局
==================

一个典型的可执行文件由以下段（Section）组成：

代码段（.text）
------------------

存放可执行指令，通常是只读的：

.. code-block:: none

   section .text
       global _start
   _start:
       mov rax, 60
       xor rdi, rdi
       syscall

已初始化数据段（.data）
---------------------------

存放有初始值的全局变量和常量：

.. code-block:: none

   section .data
       msg  db 'Hello', 0       ; 字符串常量
       val  dq  42               ; 64 位整数
       arr  dd  1, 2, 3, 4       ; 32 位整数数组

未初始化数据段（.bss）
-------------------------

存放零初始化的全局变量，**不占用可执行文件空间**：

.. code-block:: none

   section .bss
       buffer  resb  4096        ; 4K 缓冲区
       counter resq  1           ; 64 位计数器

栈
---

栈是程序运行时自动分配的内存区域，用于函数调用、局部变量和临时数据。栈从高地址向低地址增长。

堆
---

堆用于动态内存分配，在 Linux 中通过 ``brk`` 或 ``mmap`` 系统调用管理。堆从低地址向高地址增长。

字节序（Endianness）
=======================

x86_64 使用 **小端序 （Little-Endian）**。这意味着多字节数据的 **最低有效字节（LSB）存放在最低地址**。

.. code-block:: none

   ; 存储 0x12345678 到内存地址 0x1000
   mov dword [0x1000], 0x12345678

   ; 内存布局（小端序）：
   ;   地址     值
   ;   0x1000:  0x78  (LSB)
   ;   0x1001:  0x56
   ;   0x1002:  0x34
   ;   0x1003:  0x12  (MSB)

.. code-block:: none

   ; 小端序的实际影响——类型转换
   section .data
       val dq 0x0102030405060708

   section .text
       mov rax, [val]       ; rax = 0x0102030405060708
       mov al,  [val]       ; al  = 0x08（LSB 在最低地址）
       mov ah,  [val+1]     ; ah  = 0x07

.. note::

   小端序在 x86_64 上是硬件决定的，编程时通常不需要手动关心字节序——除非你正在做网络编程（网络字节序为大端序），
   或从文件中读取跨平台二进制数据。此时可以使用 ``bswap`` 指令进行大小端转换。

对齐（Alignment）
====================

CPU 访问对齐的数据效率更高。所谓对齐，是指数据的地址是其大小的整数倍。

.. list-table::
   :header-rows: 1

   * - 数据类型
     - 大小
     - 对齐要求
   * - 字节
     - 1
     - 1（总是对齐）
   * - 字
     - 2
     - 2 的倍数
   * - 双字
     - 4
     - 4 的倍数
   * - 四字
     - 8
     - 8 的倍数
   * - 双四字
     - 16
     - 16 的倍数

.. code-block:: none

   ; 对齐示例
   section .data
       align 8                   ; 确保下一个数据按 8 字节对齐
       val dq 0x12345678

       align 16                  ; SIMD 数据需要 16 字节对齐
       vec dd 1.0, 2.0, 3.0, 4.0

   ; 访问未对齐数据的代价
   ; 以下代码虽然能工作，但性能可能显著下降
   mov rax, [rbx+1]    ; rbx+1 不是 8 字节对齐 → 可能的性能惩罚

.. warning::

   对齐对性能影响巨大。对齐的内存访问是原子的（单次总线事务），而未对齐访问可能需要两次总线事务。
   在涉及 SIMD 指令时，某些指令（如 ``movaps``）**强制要求对齐**，使用未对齐版本 ``movups`` 会慢一些。
